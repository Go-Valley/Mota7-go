# إعداد إشعارات ntfy (مُتاح)

## 1) تغيير اسم الموضوع (Topic)

في `src/environments/environment.ts` و `environment.prod.ts` غيّر:

- `ntfy.topic` إلى سلسلة **طويلة وعشوائية** لا يشاركها إلا تطبيقك (مثل: `mota7-ads-xK9mP2qL...`).

أي شخص يعرف نفس الـ topic يمكنه إرسال أو استقبال الرسائل على `ntfy.sh`.

## 2) تعطيل الإشعارات العامة

اضبط `ntfy.enabled: false` — سيظل **إشعار صاحب الإعلان المحلي** يعمل عند النشر (بدون ntfy للآخرين).

## 3) بناء أندرويد

```bash
npm run build:android
```

أو: `ng build` ثم `npx cap sync android` وفتح المشروع في Android Studio.

## 4) كيف يعمل

- عند **نشر إعلان جديد**: طلب `POST` إلى `https://ntfy.sh/{topic}` مع عنوان «إعلان جديد» وجسم يتضمن معاينة الإعلان.
- التطبيق يفتح **SSE** على نفس الموضوع عند التشغيل (أندرويد فقط) ويعرض **إشعاراً محلياً** عند وصول رسالة.
- سطر `UID:...` في الجسم يمنع ظهور «إعلان جديد» لنفس الناشر على جهازه (لأنه يتلقى إشعار «تم النشر» منفصلاً).

## 5) خادم خاص (اختياري)

غيّر `ntfy.baseUrl` إلى عنوان خادم ntfy الذي تستضيفه بنفسك.

## 6) طلبات العملاء (مقدمو الخدمة)

- `ordersEnabled: true` ينشر طلبات التوصيل / التعليم / الخدمات الأخرى على ntfy بنفس `topic` ما لم تضبط `ordersTopic` منفصلاً.
- المستمع يفلتر حسب `delivery_match_key` / `education_match_key` / `other_match_key` كما في صفحة «طلبات العملاء».

## 7) صوت الإشعار (أندرويد)

انسخ `mota7.mp3` إلى `android/app/src/main/res/raw/mota7.mp3` (أو ضع الملف في `src/assets` ثم انسخه يدوياً إلى `res/raw`). قناة `mota7-orders` تستخدم هذا الملف.
